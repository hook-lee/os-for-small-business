## ✅ 출시됨 (현재 운영 중)

### Phase 1 — Foundation (24-05 ~ 25-12, archived spec)
- 단일 사용자 워크스페이스 (라파)
- 부가세/종소세 시뮬레이터
- 카테고리 정규화
- 청년창업감면 토글

### Phase 2 — 운영 도구 (26-01 ~ 26-04)
- 회원·강사·수강권·수업·급여
- 그룹 세션 + 예약
- 회원 토큰 페이지
- 메시지 발송
- 분석 페이지

### Phase 3 — SaaS 전환 (26-05)
- ✅ Supabase Auth + 회원가입
- ✅ Multi-tenant (14 테이블 owner_id + RLS)
- ✅ 보안 헤더 + rate limit + CSP
- ✅ AI 비서 (Gemini Function Calling, 7 tools)
- ✅ 월별 요약 대시보드 (Excel 우측 패널 재현)
- ✅ 세금 페이지 강화 (간이/일반 전환 + 1억800만 임계점)
- ✅ Onmove 브랜딩 + 사용자별 워크스페이스 이름

## 🚧 다음 Phase (Phase 4) — 후보

⬜ **사용자 우선순위 결정 필요**. 다음 중 1-3개만 골라서 다음 wave로:

### A. 사용자 onboarding 강화 (Activation 개선)
- 가입 직후 3단계 wizard (회원 추가 → 수강권 카탈로그 → 첫 거래)
- 빈 화면에 "여기 클릭부터 시작" 안내
- 샘플 데이터 옵션 (5명 demo)
- ⚙ 예상 ROI: 가입 → 활성 사용자 전환율 ↑

### B. 자동 카테고리 추천 (AI)
- 거래 입력 시 거래처 + 금액 보고 카테고리 자동 추천
- "쿠팡 14,000원" → "소모품 92%" 칩 표시
- ⚙ 예상 ROI: 입력 속도 ↑ + 정확도 ↑

### C. 영수증 OCR
- 사진 업로드 → 거래 자동 입력
- Gemini Vision 또는 Google Vision API
- ⚙ 예상 ROI: 입력 마찰 대폭 감소 (가장 큰 페인)

### D. 회원 알림 자동화
- 만료 임박 회원에게 자동 카톡/SMS
- 노쇼 후 안내 메시지
- ⚙ 추가 요구: 카톡 비즈니스 채널 또는 SMS 게이트웨이 통합

### E. 강사 권한 분리
- 강사 본인 로그인 → 자기 수업/회원만
- 강사용 모바일 친화 페이지
- ⚙ 다른 SaaS의 표준. 라파처럼 1인 운영이면 우선순위 낮음.

### F. 이상치 감지 + push 알림
- 매일/주간 cron으로 데이터 검사
- "이번 주 노쇼 +50% 감지" 같은 능동 알림
- AI가 사장님에게 먼저 말 거는 형태
- ⚙ 차별점 극대화

### G. 다음 분기 예측 / 시즌성
- 24-25 데이터로 시즌성 학습 → 다음 분기 매출 예측
- 6월 여름 이벤트 / 11월 최저점 패턴
- ⚙ 진짜 가치 있지만 데이터량 따라 정확도 달라짐

### H. PWA / 모바일 최적화
- 사장님이 핸드폰으로 거래 입력
- 홈 화면 추가 가능 (PWA install)
- ⚙ 현재도 반응형이지만 모바일 first UX는 아님

### I. 강사 급여 자동 송금
- 정산 → 카카오뱅크/토스 결제 API 연동
- 클릭 한 번에 송금
- ⚙ 결제 라이센스 필요. 복잡함

### J. 외부 회계 연동
- 홈택스 API → 카드 매입 자동 import
- 사업자 카드 거래가 자동으로 transactions에 들어옴
- ⚙ 가장 큰 가치 + 가장 큰 복잡도

## 📅 단기 (1-3개월) — ⬜ 결정 필요

⬜ 위 A~J 중 어떤 거 우선?

추천 조합 (제안):
- **첫 wave**: A (onboarding) + B (카테고리 추천) — 가장 작고 임팩트 큼
- **두 번째**: C (OCR) — 가장 차별화
- **세 번째**: F (이상치 감지) — AI 가치 극대화

## 🔭 장기 (3-12개월) — ⬜ 결정 필요

- 가격 정책 도입 (Freemium / Flat / 사용량 기반)
- 외부 사용자 확장 — 어떤 사장님부터?

- 외부 API 연동 (홈택스, 카드사, 카카오)
- 모바일 앱 (React Native?)
- 결제 통합 (PG 연동)

### 인프라 ⬜
- 데이터 백업 자동화 (Supabase Free 한계)
- Upstash Redis (rate limit 분산)
- Sentry (에러 모니터링)
- Posthog/Mixpanel (사용 분석)

## 🚫 안 할 것 (의도적)

- 다른 자영업 종목 확장 (미용/카페 등) — 운동 도메인 깊이 우선
- 데스크탑 앱 — 웹 우선
- 회원 결제를 우리가 처리 (PG 라이센스 회피) — 향후 검토
- 강사 평가/리뷰 시스템 — 산업 특성상 갈등 유발
- 글로벌 확장 — 한국 세법 특화로 깊이 우선

## 의사결정 프로세스 (다음 작업 시작 전)

1. **새 요구가 들어왔는가?** → 위 후보 리스트에 추가
2. **현재 사용자(라파) 페인을 직접 해결하는가?** → 우선
3. **차별화에 기여하는가?** (다른 SaaS가 못 함) → 우선
4. **multi-tenant 격리 / 보안 / 데이터 정합성 영향 있는가?** → 신중
5. **만들고 나서 사용자가 안 쓰면?** — 작게 시작, A/B 또는 feedback loop
