# Onmove — User Flows

> **목적**: 사장님이 우리 제품을 *어떻게 쓰는지* 시나리오로 정리. 새 페이지 만들 때 "이게 어떤 flow에 들어가나" 확인.

⬜ = 사용자가 추가/수정해야 할 부분

---

## Flow 1. 신규 사용자 가입 → 첫 데이터 입력 (Activation)

> 가장 중요한 flow. 여기서 막히면 retention 0.

```
랜딩 / 회원가입 안내
    ↓
[/signup] 이메일/비번 + 센터명/직급 입력
    ↓
가입 즉시 자동 로그인 + /api/profile 자동 저장
    ↓
[/] 홈 — 빈 워크스페이스
    ↓
첫 거래 입력 [/add]
  - 5월 1일 매출 +500,000원 카드 박지영 회원 (개인 30회 수강권)
    ↓
자동: transactions row + passes row 동시 생성
    ↓
[/sales] 매출 리포트에 즉시 반영
[/finances] 월별 요약에도
[/members] "박지영" 회원 자동 등록
```

**아쉬운 점 / ⬜ 개선**:
- 첫 가입 시 onboarding wizard 없음. 빈 화면 보고 어디서 시작할지 막막할 수 있음
- ⬜ 가입 직후 onboarding tour 추가? (3단계: 회원 추가 → 수강권 카탈로그 → 거래 입력)
- ⬜ 샘플 데이터로 시작하는 옵션? (5명 회원 demo 데이터, 사용자가 지우든 익히든)

## Flow 2. 회원 결제 → 수강권 발급 → 매출 (가장 자주 일어남)

```
회원이 "개인 30회 65만원" 결제하겠다고 함
    ↓
사장님: [/add] 거래 입력
  - 매출 토글 ON
  - 회원: "박지영" (검색 + 선택)
  - 수강권: "개인 30회 · 30회/365일 · 650,000원" (자동 정가 입력)
  - 결제수단: 카드
  - 결제 구분: 신규결제 / 재결제 (사장님 선택)
    ↓
저장 클릭 → 자동:
  - transactions.insert (매출 row, owner_id 격리)
  - passes.insert (수강권 발급, 잔여 30회)
    ↓
즉시 반영:
  - [/finances] 이번 달 매출 +65만
  - [/members/[id]] 박지영 활성 수강권 1개
  - 회원 토큰 페이지 [/m/[token]] 자동 갱신
  - AI 비서 "박지영 활성 수강권?" 답변에 즉시 반영
```

**현재 잘 됨**. ⬜ 개선 가능:
- 카드 결제 시 PG사와 통합해서 자동 입력? (스튜디오메이트 비교 우위)
- 영수증 사진 → OCR → 자동 입력?

## Flow 3. 수업 등록 → 차감 (회원/강사 운영)

```
박지영 회원이 5/15 10시 개인 수업 예약
    ↓
[/lessons] → 날짜 선택 → "+ 수업 추가"
  - 회원: 박지영 → 활성 수강권 자동 매칭
  - 강사: 김유진
  - 시간: 10:00
    ↓
lessons.insert (status='scheduled', pass_id, owner_id)
    ↓
[당일] 수업 끝나면 사장님이 status 변경: scheduled → completed
    ↓
자동 차감:
  - passes.remaining_count -= 1
  - 만약 잔여 0 + 이용중이면 → status='이용만료' 전환
  - lessons.deducted = true
  - (AI 비서가 다음 번에 "박지영 활성 수강권?"하면 잔여 29회 답변)
    ↓
[월말] payroll_records 강사 급여 자동 집계
  - 김유진 5월 개인 수업 X회 × 32,000원 = 시급
  - 3.3% 원천징수
```

**현재 잘 됨**. ⬜ 개선:
- 회원이 직접 [/m/[token]/lessons]에서 예약? 그러면 사장님은 승인만?
- 노쇼 자동 처리 (수업 시간 끝나면 자동 noshow)

## Flow 4. 월말 정산 (매월 1일 사장님 의식)

```
[/finances] 월별 요약 → 지난달 선택
    ↓
4 KPI 확인: 매출 / 영업이익 / 순수익 / 거래 건수
    ↓
좌측 사업 비용 9개 점검
  - 임대료 / 관리비 정확한지
  - 정기결제 / 보험료 변동 있는지
    ↓
우측 개인 비용 12개 점검 (대표자 personal 영역)
    ↓
영업이익 vs 순수익 차이 확인
    ↓
[/sales] → 매출 source: 카드/계좌이체/현금 비율 확인
    ↓
[/payroll] 강사 급여 정산
  - 강사별 자동 집계 결과 확인
  - 3.3% 원천 공제 확인
  - 정산 완료 마킹
    ↓
[/tax] 다음 분기 부가세 예상 확인 → 예비비 조정
```

**현재 잘 됨**. ⬜ 개선:
- 월말 자동 알림? (1일 아침 "지난달 정산 시작" cron)
- 강사 급여 자동 송금 통합?

## Flow 5. 세금 의사결정 (분기/연 단위)

```
부가세 분기 신고 1개월 전 (3/31, 6/30, 9/30, 12/31 직전)
    ↓
[/tax] 진입
    ↓
**섹션 1**: 사업자 유형 타임라인 확인
  - "지금 일반과세자, 26-07부터 간이 전환 가능 조건부"
    ↓
**섹션 2**: 간이 전환 조건 모니터링
  - rolling 12개월 매출 vs 1억800만
  - 만약 초과 중 → "이번 분기 매출 의도적으로 줄일 옵션?"
  - 만약 미만 → "26-07부터 간이 전환 가능. 신청은 6월 말까지"
    ↓
**섹션 3**: 분기별 부가세 추이
  - 이번 분기 추정 = X원 → 예비비 충분한지 확인
    ↓
**섹션 4**: 과거 실제 vs 예상
  - 추정 정확도 확인
    ↓
**하단**: 종소세 시뮬레이터
  - 연환산 매출 → 사업소득 → 과세표준 → 예상 납부액
  - 청년창업감면 적용 여부 확인
    ↓
의사결정:
  - 예비비 부족 → 다음 달 예비비 적립 늘림
  - 간이 전환 가능 시점 캘린더 표시 → 6월 신청 알림
  - 절세 액션 (광고비 영수증 챙기기 등)
```

⬜ **개선 가능**:
- 신고일 D-day 카운트다운 + 자동 알림
- "이번 분기 매출 X원 줄이면 간이 가능" 시뮬레이션
- AI 비서가 절세 액션 추천

## Flow 6. AI 비서 상담 (Ad-hoc)

```
사장님 머리에 "이번 달 광고비 얼마 썼지?" 떠오름
    ↓
어떤 페이지에 있든 우하단 💬 클릭
    ↓
챗 widget 열림 — 현재 페이지 컨텍스트 자동 인지
("재무 화면을 보고 계시네요" 메시지)
    ↓
"이번 달 광고비 얼마 썼어?"
    ↓
Gemini Function Calling:
  - getMonthlyFinancials(year=2026, month=5)
  - expensesByCategory에서 광고선전비 추출
    ↓
답변: "5월 광고선전비 X원 (총 N건). 가장 큰 건은 ..."
    ↓
🔧 tool 호출 라벨 표시 → 사장님이 어떤 데이터 봤는지 검증 가능
    ↓
대화 자동 영속화 (Supabase chat_sessions)
    ↓
다음에 다른 기기에서 로그인해도 같은 대화 이력
```

⬜ **개선**:
- AI가 능동적으로 추천도? "이번 달 광고비 +400%, 이상치 감지" cron 알림
- write 권한 추가? "박지영 메모에 '이번주 노쇼' 추가해줘"

## Flow 7. 회원 페이지 (회원 입장)

```
사장님이 [/members/[id]] → 회원 토큰 URL 복사
    ↓
카톡으로 회원에게 전송 "수강권 현황 확인"
    ↓
회원 클릭 → [/m/[token]] (로그인 X, 토큰만으로 접근)
    ↓
홈: 현재 수강권 잔여 X회 / 만료 Y일 남음
    ↓
탭 이동:
  - 수강권: 전체 이력
  - 수업: 다가오는 수업
  - 그룹: 예약 가능한 그룹 세션
    ↓
[/m/[token]/group] 그룹 예약
  - 회원이 직접 예약 → group_reservations.insert
  - 사장님 [/lessons/groups/[id]] 명단에 자동 반영
```

⬜ **개선 가능**:
- 회원이 결제 직접? (PG 통합)
- 회원에게 알림 (수강권 만료임박 / 노쇼 후 안내)
