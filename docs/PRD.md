## 1. 비전 / 미션 / 한 줄 정의

### 비전
> "운동 센터 운영의 모든 결정을 데이터 기반으로. 사장님은 결정만 하고, AI 비서가 자기 데이터를 들고 답한다."

### 미션
필라테스/PT/요가 1인 운영자에게 **회원·강사·매출·세금**을 통합 관리할 단일 워크스페이스 + AI 상담을 제공해, 세무사·회계사·외주 ERP 없이도 의사결정 가능하게 한다.

### 한 줄 정의
> 운동 센터 사장님이 회원·강사·매출·세금을 한 곳에서 굴리고, AI 비서가 자기 데이터를 들고 상담해주는 SaaS.

## 2. ICP (Ideal Customer Profile)

### 1차 ICP (현재 운영 중)

**유진** — 라파 필라테스 원장
- 만 27세, 2024-04 창업, 1인 운영
- 연 매출 1~1.2억대 (24년 1.02억, 25년 1.18억)
- 24-04~25-06 간이과세자 → 25-07~ 일반과세자 (강제 전환) → 26-07~ 간이 전환 검토 중
- 이전 도구: 구글 시트 + Looker Studio + 스튜디오메이트
- 세무사: 5월 종소세 신고 때만 접촉. 평소 세금 정보 부재
- 페인:
  - 분기 부가세 신고 직전에야 금액 확인
  - 강사 급여 정산 매월 수동 계산 (3.3% 원천징수까지)
  - 회원 결제 + 수강권 발급 + 매출 입력을 3군데에 따로 입력
  - 1억800만 임계점 모니터링 불가 → 간이↔일반 전환 시점 놓침

## 4. 핵심 기능 (현재 운영 중)

### A. 인증 / Multi-tenant ✅
- Supabase Auth, RLS, 회원가입(센터명·직급 필수)
- 회원 토큰 페이지 (인증 우회)

### B. 거래 가계부 ✅
- `/add` 한 줄 입력, 카테고리 24개 default + 사용자 추가
- 회원·강사·수강권 같이 선택 시 → 자동 수강권 발급

### C. 회원·강사·수강권 ✅
- 회원 285명 (라파 기준), 강사 시급 4종, 수강권 자동 차감

### D. 수업 ✅
- 개인/그룹 수업, 반복 등록, 차감 룰

### E. 강사 급여 ✅
- 자동 집계 + 3.3% 원천징수

### F. 재무 ✅
- 월별 요약 / 매출 / 세금 / 분석 / 카테고리

### G. AI 비서 ✅
- Gemini Function Calling, 7 tools, 영속화

### H. 카테고리 가이드 ✅
- 24개 표준 회계 계정과목

## 5. 차별점 / 경쟁

| | 스튜디오메이트 | 일반 가계부 SaaS | 세무 SaaS | **Onmove** |
|---|:---:|:---:|:---:|:---:|
| 회원·수강권 관리 | ✓ | X | X | ✓ |
| 매출·지출 가계부 | △ | ✓ | △ | ✓ |
| 세금 시뮬레이터 | X | △ | ✓ | **✓** |
| 간이↔일반 전환 모니터링 | X | X | △ | **✓** |
| 강사 급여 자동 집계 | △ | X | X | **✓** |
| AI 비서 (데이터 들고 상담) | X | X | X | **✓** |
| Multi-tenant 무료 | X | △ | X | **✓** (현재) |

**핵심 차별점 한 줄**: 다른 SaaS는 데이터를 *보여주기만* 한다. Onmove는 사장님이 *물어보면* AI가 그 데이터를 들고 답한다.

⬜ **결정 필요**: 정확한 경쟁사 리스트 (스튜디오메이트 외에 누구를 의식하나?)

## 6. 성공 지표 (KPI)

⬜ **결정 필요** — 후보:

### 사용자 단위
- DAU (Daily Active Users)
- 평균 거래 입력 빈도 (주당 N회)
- AI 비서 사용 빈도 (사용자당 / 주)
- 세금 페이지 방문 횟수 (간이 전환 결정에 기여하는지)

### 사업 단위
- MAU (Monthly Active Users)
- 가입 → 첫 거래 입력까지 시간 (activation)
- 30일 retention
- ⬜ 유료 전환율 (paid 도입 시)

### 정성
- "이거 없으면 못 산다" 응답 비율
- NPS

## 7. 가격 정책

⬜ **현재 무료** (Vercel Hobby + Supabase Free + Gemini Free) — 사용자 늘면 한계 도달

### 향후 옵션 (결정 필요)

- (A) **계속 무료** — 라파 본인 + 친한 사장님 몇 명만
- (B) **Freemium**
  - Free: 회원 50명 / AI 비서 월 100회 / 1년 데이터
  - Pro: $19/월 — 무제한
- (C) **Flat 유료** — 모든 사용자 $10/월
- (D) **이용량 기반** — AI 호출 + 거래 건수에 비례

각 옵션별 트레이드오프:
- (A): 사업 X, 유진 선물용 그대로
- (B): 확장 가능, 수익 발생, 다만 freemium 한계 설계 필요
- (C): 단순, retention 강력 필요
- (D): 가장 fair하지만 사용자 입장에선 예측 어려움

## 8. 의사결정 원칙

1. **데이터 source of truth는 사용자 직접 입력** — 외부 import는 참고용
2. **세무 답변엔 항상 디스클레이머** — "참고용 시뮬레이션, 신고는 세무사 확인"
3. **새 사용자는 빈 워크스페이스로 시작** — fixture/demo 자동 import X
4. **DB 마이그레이션은 멱등** — 여러 번 실행 안전
5. **사용자 결정 권한**: 데이터 삭제·실제 신고·결제 — 모두 사용자가 클릭해야 적용
6. **yes-man 금지** — 약점·리스크 먼저 짚기
7. **multi-tenant 격리는 절대 양보 X** — 코드 + DB(RLS) 이중 보호
8. **AI 비서는 read-only** — 데이터 수정/삭제 권한 X
